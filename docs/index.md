# BeyondTrust Provider

The BeyondTrust provider allows you to manage BeyondTrust resources (Workload Credentials and other BeyondTrust services) using infrastructure as code.

## Authentication

The provider authenticates using a **personal access token** issued by the BeyondTrust Pathfinder Platform.

To obtain credentials:

1. Log in to [BeyondTrust Pathfinder](https://app.beyondtrust.io)
2. Navigate to **User Settings → Manage Profile → Personal Access Tokens**
3. Create a token and copy it — you will not be able to retrieve it again

The provider also requires your **Site ID** (a UUID identifying your tenant) and the **API URL** for your Workload Credentials instance. Contact your BeyondTrust administrator if you do not have these values.

### Environment Variables (Recommended)

Set credentials via environment variables to keep them out of your Terraform configuration files:

```shell
export BEYONDTRUST_ACCESS_TOKEN="your-token"
export BEYONDTRUST_SITE_ID="a1b2c3d4-e5f6-7890-abcd-ef1234567890"
```

With environment variables configured, the provider block needs no explicit credentials:

```hcl
provider "beyondtrust" {}
```

### Explicit Configuration

For non-sensitive values you may set them directly. Always use Terraform variables or a secrets manager for the access token — never hard-code it:

```hcl
provider "beyondtrust" {
  api_url      = "https://api.beyondtrust.io"
  access_token = var.beyondtrust_access_token
  site_id      = var.beyondtrust_site_id
}
```

## Example Usage

```terraform
terraform {
  required_version = ">= 1.11.0"

  required_providers {
    beyondtrust = {
      source  = "beyondtrust/beyondtrust"
      version = "~> 1.0"
    }
  }
}

# Credentials are read from environment variables:
#   BEYONDTRUST_ACCESS_TOKEN
#   BEYONDTRUST_SITE_ID
provider "beyondtrust" {}

# Create a folder to organize secrets
resource "beyondtrust_workload_credentials_folder" "example" {
  name = "production"

  tags = {
    env     = "production"
    team    = "platform"
    managed = "terraform"
  }
}
```

### Optional

- `access_token` (String, Sensitive) The API access token for authentication. Can also be set via BEYONDTRUST_ACCESS_TOKEN environment variable.
- `api_path_version` (String) Optional API path version (e.g., 'v1'). Defaults to empty string (no path version). Can also be set via BEYONDTRUST_API_PATH_VERSION environment variable.
- `api_url` (String) The base URL for the BeyondTrust API. Defaults to https://api.beyondtrust.io; override for other deployments such as GovCloud. Can also be set via BEYONDTRUST_API_URL environment variable.
- `api_version` (String) The API header version (date-based, e.g., '2026-04-28'). Defaults to '2026-04-28'. Can also be set via BEYONDTRUST_API_VERSION environment variable.
- `insecure` (Boolean) Skip TLS certificate verification. Only use for development. Defaults to false. Can also be set via BEYONDTRUST_INSECURE environment variable.
- `role` (String) Role for X-BT-Role header (when set, X-BT-Auth-Type is automatically set to 'CUSTOM-IDP'). Can also be set via BEYONDTRUST_ROLE environment variable.
- `service_name` (String) Service name for GitHub OIDC authentication (X-BT-Service-Name header). Required when using GitHub OIDC tokens. Can also be set via BEYONDTRUST_SERVICE_NAME environment variable.
- `site_id` (String) The site/tenant ID (UUID format). Required for multi-tenant deployments. Can also be set via BEYONDTRUST_SITE_ID environment variable.
- `timeout` (String) HTTP client timeout duration (e.g., '30s', '1m'). Defaults to '30s'. Can also be set via BEYONDTRUST_TIMEOUT environment variable.
